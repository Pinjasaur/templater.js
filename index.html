templater.js

A tiny JS library for minimal {{templating}}.

Download → GitHub Project →

tl;dr

templater.js is a teeny JavaScript library for minimal templating needs (soon to be made obsolete by ES6 templating). Its templating expression syntax mimics that of {{mustache}}/Handlebars. Barebones conditional logic included.

Features

  • No eval()s or new Function() { … }s
  • HTML characters (<, >, and &) automatically escaped
  • < 1KB after minification
  • Zero dependencies

Limitations

  • Cannot nest logic statements
  • Cannot nest properties (e.g. {{foo.bar.baz}})

Getting Started

  1. Add the script to your HTML document: 
    <script src="path/to/templater.min.js"></script>
  2. Create the template you'd like to use:
    3. <!-- by using a non-standard `type` the browser won't parse it -->
<script id="my-template" type="text/template">
  <h2>Hi, my name is {{ name }}.</h2>
  <p>I am {{ age }} years old and live in {{ location }}.</p>
</script>
  3. Parse the template against some data and render it to the DOM: 
    var template = templater(document.getElementById("my-template").innerHTML);

document.body.insertAdjacentHTML("beforeend", template({
  name: "John",
  age: 27,
  location: "The United States"
});

Syntax

Expressions

Syntax mimics {{mustache}}/Handlebars: {{ contents }} where contents represents what you'd like to replace when the template is rendered against some data. Note that the amount of whitespace between contents and the two braces doesn't matter. For example, the following are all equivalent:

  • {{contents}}
  • {{    contents}}
  • {{contents   }}
  • {{   contents    }}

However, for the sake of readability I would recommend using 0 or 1 spaces between the braces and the expression body.

Conditionals

You can use a #if directive to conditionally render template contents based on the truthy value of the arguments. The syntax looks like:

{{ #if (condition) }}
  <!-- this only renders if `condition` evaluates to a truthy value -->
  
My conditional contents: {{ contents }}

{{ /if }}

To provide multiple arguments to the condition, just separate them by commas: {{ #if (foo, bar, baz) }}

For example, given the following template snippet

{{ #if (name, age) }}
  <p>Hi, my name is {{ name }} and I am {{ age }} years old.</p>
{{ /if }}

rendered against the following

{
  name: "John",
  age: 27
}

will produce

<p>Hi, my name is John and I am 27 years old.</p>

however, rendered against data where name or age is falsy

{
  name: "John",
  age: 0
}

the body of the #if condition won't be rendered.

Limitations

Nested #ifs

Due to how the templating is handled (regular expressions) nested #if statements will not work. The script won't complain, but the desired logic will not be achieved.

Nested properties

Once again, due to how the replacement in the templating is done, nested properties will not work; {{ foo.bar.baz }} will look for a property named "foo.bar.baz" not the property baz of the property bar of the property foo.
